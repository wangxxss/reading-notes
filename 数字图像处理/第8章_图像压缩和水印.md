# 第8章 图像压缩和水印

## 学习目标

- 理解图像压缩的基本原理
- 掌握有损压缩和无损压缩的区别
- 了解常见的压缩算法（JPEG等）
- 理解数字水印的概念和应用
- 掌握图像压缩标准的发展

---

## 8.1 基本概念

### 8.1.1 图像压缩的必要性

**数据量示例**：
| 图像类型 | 分辨率 | 位深度 | 原始大小 |
|----------|--------|--------|----------|
| 标清电视 | 720×480 | 24bit | ~1MB/帧 |
| 1080P | 1920×1080 | 24bit | ~6MB/帧 |
| 4K视频 | 3840×2160 | 24bit | ~25MB/帧 |
| 2小时电影(未压缩) | - | - | ~224GB |

**压缩的价值**：
- 节省存储空间
- 减少传输带宽
- 降低成本

### 8.1.2 压缩的基本原理

**压缩的本质**：
```
用尽可能少的数据表达尽可能多的信息
```

**信息度量**：
```
信息量 I(x) = -log₂ P(x)
比特数 = 信息量的期望值
```

### 8.1.3 压缩系统模型

```
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│ 原始图像  │ → │ 变换器   │ → │ 量化器   │ → │ 编码器   │ → │ 压缩数据 │
└──────────┘    └──────────┘    └──────────┘    └──────────┘
                     ↑                           ↓
                     └──── [量化控制] ←────────┘
```

**解码过程**：
```
┌──────────┐    ┌──────────┐    ┌──────────┐    ┌──────────┐
│ 压缩数据 │ → │ 解码器   │ → │ 反量化   │ → │ 反变换   │ → │ 重建图像 │
└──────────┘    └──────────┘    └──────────┘    └──────────┘
```

## 8.2 图像冗余

### 8.2.1 编码冗余

**定义**：用于表示图像的码字长度与图像信息熵不匹配

**信息熵**：
```
H = -Σ p(aᵢ) log₂ p(aᵢ)
```

**最优码长**：接近熵 H

**编码效率**：
```
η = H / L̄ × 100%
其中 L̄ 为平均码长
```

### 8.2.2 空间冗余（像素间冗余）

**定义**：同一行/列相邻像素间存在相关性

**例子**：
- 平滑区域像素值相近
- 边缘附近像素值渐变

**利用方式**：
- 预测编码
- 变换编码

### 8.2.3 时间冗余（视频）

**定义**：相邻帧之间的相关性

**利用方式**：
- 帧间预测
- 运动补偿

### 8.2.4 心理视觉冗余

**定义**：人眼感知不到或忽略的信息

**特点**：
- 某些灰度变化人眼不可感知
- 与视觉特性相关
- 去除不影响视觉效果

## 8.3 无损压缩

### 8.3.1 基本原理

**定义**：压缩后可完全恢复原始数据

**特点**：
- 无信息损失
- 压缩比有限（通常2:1-10:1）
- 用于需要精确重建的应用

### 8.3.2 变长编码

**霍夫曼编码**：
- 使用变长码
- 高频符号短码
- 低频符号长码
- 前缀码特性

**香农-范诺编码**：
- 基于概率分配码字
- 平均码长接近熵

**算术编码**：
- 将整个序列编码为一个数
- 效率高于霍夫曼
- 无需整数比特

### 8.3.3 LZW编码

**原理**：基于字典的压缩

**过程**：
1. 初始化字典（包含单字符）
2. 扫描输入，查找最长匹配串
3. 输出字典索引
4. 添加新串到字典

**应用**：
- GIF格式
- TIFF格式
- UNIX compress

### 8.3.4 行程编码 (RLE)

**原理**：记录连续重复值的次数

**示例**：
```
原始: A A A A B B C C C D D D D
编码: 4A 2B 3C 4D
```

**适用场景**：
- 二值图像
- 简单灰度图像
- 打印图像

---

## 8.4 有损压缩

### 8.4.1 基本原理

**定义**：压缩后无法完全恢复原始数据

**特点**：
- 压缩比高
- 允许一定失真
- 广泛应用于图像/视频

### 8.4.2 量化

**标量量化**：
```
连续值 → 离散值
量化等级 L
```

**矢量量化**：
```
向量空间 → 码书索引
```

### 8.4.3 预测编码

**基本思想**：
- 不编码像素值本身
- 编码预测误差

**差分脉冲编码调制 (DPCM)**：
```
e = f - f̂
编码 e 而非 f
```

### 8.4.4 变换编码

**JPEG基本步骤**：
1. 图像分块（8×8）
2. DCT变换
3. 量化
4. 熵编码

## 8.5 JPEG压缩

### 8.5.1 JPEG标准概述

**JPEG** = Joint Photographic Experts Group

**压缩模式**：
| 模式 | 特点 |
|------|------|
| 顺序编码 | 从上到下，从左到右 |
| 渐进编码 | 多次扫描，逐渐清晰 |
| 无损编码 | 预测编码，完美重建 |
| 分层编码 | 多分辨率 |

### 8.5.2 JPEG压缩步骤

```
┌─────────────────────────────────────────────────────┐
│ 1. 颜色转换 (RGB → YCbCr)                          │
└─────────────────────────────────────────────────────┘
                         ↓
┌─────────────────────────────────────────────────────┐
│ 2. 分块 (8×8)                                      │
│     [ 8  8  8  8  8  8  8  8 ]                     │
│     [ 8  8  8  8  8  8  8  8 ]                     │
│     ...                                            │
└─────────────────────────────────────────────────────┘
                         ↓
┌─────────────────────────────────────────────────────┐
│ 3. DCT变换                                         │
│     f(x,y) → F(u,v)                                │
└─────────────────────────────────────────────────────┘
                         ↓
┌─────────────────────────────────────────────────────┐
│ 4. 量化 (关键步骤，产生失真)                        │
│     使用量化表 Q                                    │
│     Fq(u,v) = round(F(u,v) / Q(u,v))               │
└─────────────────────────────────────────────────────┘
                         ↓
┌─────────────────────────────────────────────────────┐
│ 5. Z字形扫描                                       │
│     [0] [1] [5] [...]                               │
│     将二维系数转为一维序列                          │
└─────────────────────────────────────────────────────┘
                         ↓
┌─────────────────────────────────────────────────────┐
│ 6. 直流系数(DC)差分编码                             │
│     DCᵢ = DCᵢ - DCᵢ₋₁                             │
└─────────────────────────────────────────────────────┘
                         ↓
┌─────────────────────────────────────────────────────┐
│ 7. 交流系数(AC)行程编码                            │
│     (零的个数, 非零值大小)                          │
└─────────────────────────────────────────────────────┘
                         ↓
┌─────────────────────────────────────────────────────┐
│ 8. 霍夫曼编码                                      │
└─────────────────────────────────────────────────────┘
```

### 8.5.3 量化表

**亮度量化表**：
```
[16  11  10  16  24  40  51  61
 12  12  14  19  26  58  60  55
 14  13  16  24  40  57  69  56
 14  17  22  29  51  87  80  62

 24  35  55  64  81 104 113  92
```

**特点**：
- 低频系数精细量化
- 高频系数粗糙量化
- 符合人眼视觉特性

### 8.5.4 JPEG2000

**改进**：
- 基于小波变换
- 支持无损压缩
- 更好的压缩效率
- 感兴趣区域编码

## 8.7 数字水印

### 8.7.1 基本概念

**定义**：在图像中嵌入不可见的标识信息

**目的**：
- 版权保护
- 内容溯源
- 认证

### 8.7.2 水印系统要求

| 要求 | 说明 |
|------|------|
| **不可见性** | 不影响图像视觉效果 |
| **鲁棒性** | 抗攻击（压缩、滤波、裁剪） |
| **容量** | 可嵌入的数据量 |
| **安全性** | 防止未授权提取 |

### 8.7.3 水印嵌入方法

**空间域方法**：
- 最低有效位 (LSB) 替换
- 优点：容量大
- 缺点：鲁棒性差

**变换域方法**：
- DCT系数修改
- DWT系数修改
- 优点：鲁棒性好
- 缺点：计算复杂

### 8.7.4 水印攻击

| 攻击类型 | 说明 |
|----------|------|
| 有损压缩 | JPEG压缩 |
| 滤波 | 低通/高通滤波 |
| 几何变换 | 旋转、缩放、裁剪 |
| 噪声 | 加性噪声 |
| 共谋攻击 | 多幅图像分析 |

## 本章小结

### 压缩方法分类

```
┌─────────────────────────────────────────┐
│             图像压缩                     │
├─────────────┬─────────────┬─────────────┤
│  无损压缩    │  有损压缩    │  水印       │
├─────────────┼─────────────┼─────────────┤
│ 霍夫曼编码   │ 量化        │ 空间域嵌入   │
│ 算术编码     │ 预测编码    │ 变换域嵌入   │
│ LZW编码      │ 变换编码    │ 鲁棒性分析   │
│ 行程编码     │ JPEG/JPEG2000│            │
└─────────────┴─────────────┴─────────────┘
```

### 压缩性能比较

| 方法 | 压缩比 | 质量损失 |
|------|--------|----------|
| 无损 (JPEG-LS) | 2:1 | 无 |
| JPEG | 10:1 - 20:1 | 可控 |
| JPEG2000 | 10:1 - 50:1 | 可控 |
| H.264 | 50:1 - 100:1 | 依赖质量 |

### 关键概念

1. **编码冗余**：码字长度与熵不匹配
2. **空间冗余**：像素间相关性
3. **心理视觉冗余**：人眼不敏感信息
4. **JPEG**：基于DCT的块编码
5. **水印**：鲁棒性 vs 不可见性权衡
