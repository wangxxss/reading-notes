## 1. 计算机系统组成

### 1.1 计算机硬件组成

计算机硬件系统由**运算器、控制器、存储器、输入设备、输出设备**五大部件组成。

| 部件 | 功能 | 核心组件 |
|------|------|----------|
| 运算器 | 执行算术运算和逻辑运算 | ALU（算术逻辑单元） |
| 控制器 | 协调控制各部件工作 | PC、IR、CU |
| 存储器 | 存储程序和数据 | 主存、辅存、Cache |
| 输入设备 | 输入外部信息 | 键盘、鼠标、扫描仪 |
| 输出设备 | 输出处理结果 | 显示器、打印机 |

### 1.2 计算机软件组成

```
计算机软件
├── 系统软件
│   ├── 操作系统
│   ├── 编译程序
│   ├── 汇编程序
│   └── 设备驱动程序
└── 应用软件
    ├── 办公软件
    ├── 数据库系统
    └── 网络软件
```

### 1.3 中央处理器（CPU）

CPU是计算机的核心，由**运算器、控制器、寄存器组**组成。

**运算器包含：**
- 算术逻辑单元（ALU）
- 累加寄存器（AC）
- 数据缓冲寄存器（DR）
- 状态条件寄存器（PSW）

**控制器包含：**
- 指令寄存器（IR）
- 程序计数器（PC）
- 地址寄存器（AR）
- 指令译码器（ID）

**考试重点：**
- CPU的功能是取指、执行指令
- PC存放下一条要执行的指令地址
- IR存放当前正在执行的指令

## 2. 计算机体系结构分类

### 2.1 Flynn分类法（常考）

| 分类 | 结构特点 | 典型代表 |
|------|----------|----------|
| SISD | 单指令流、单数据流 | 传统串行计算机 |
| SIMD | 单指令流、多数据流 | 并行处理器、GPU |
| MISD | 多指令流、单数据流 | 理论模型（较少实现） |
| MIMD | 多指令流、多数据流 | 多处理器系统 |

**考试重点：** Flynn分类法是最常考的分类方法，需熟记四种类型的特点。

### 2.2 冯氏分类法

按数据流和指令流分类，与Flynn类似。

### 2.3 处理器性能提升技术

| 技术 | 描述 |
|------|------|
| 时间重叠 | 流水线技术 |
| 资源重复 | 多处理器、多存储体 |
| 资源共享 | 分时系统 |

## 3. 指令系统

### 3.1 指令的基本格式

```
[操作码 OP] [地址码 A]
```

### 3.2 指令长度与地址码

| 类型 | 地址码数目 | 特点 |
|------|------------|------|
| 零地址指令 | 0 | 操作数隐含在栈顶 |
| 一地址指令 | 1 | 操作数隐含在AC |
| 二地址指令 | 2 | 常见格式：OP A1 A2 |
| 三地址指令 | 3 | 最完整形式 |

**考试重点：** 地址码数目越多，指令越长，占用存储空间越大。

### 3.3 寻址方式（重要考点）

| 寻址方式 | EA=A | 特点 |
|----------|------|------|
| 立即寻址 | 操作数在指令中 | 最快，但修改不灵活 |
| 直接寻址 | EA=A | 地址在指令中 |
| 间接寻址 | EA=(A) | 一次或多次访存 |
| 寄存器寻址 | EA=R | 速度最快 |
| 寄存器间接寻址 | EA=(R) | 操作数在内存中 |

**速度比较：** 立即寻址 > 寄存器寻址 > 直接寻址 > 寄存器间接寻址 > 间接寻址

### 3.4 CISC与RISC（常考对比）

| 特征 | CISC | RISC |
|------|------|------|
| 指令条数 | 多（>200） | 少（<100） |
| 指令长度 | 变长 | 定长 |
| 执行时间 | 不等 | 大多单周期 |
| 寻址方式 | 多种 | 少 |
| 通用寄存器 | 少 | 多 |
| 译码方式 | 微程序控制 | 硬连线控制 |
| 负载均衡 | 编译器复杂 | 编译器简单 |

**代表架构：**
- CISC：x86（Intel）、VAX
- RISC：ARM、MIPS、PowerPC

**考试重点：** RISC的关键技术包括寄存器窗口优化、流水线优化、延迟加载等。

## 4. 存储系统

### 4.1 存储器的层次结构

```
┌─────────────────────────────────────┐
│           寄存器 (CPU内部)           │ ← 速度最快，容量最小
├─────────────────────────────────────┤
│           Cache (SRAM)              │ ← 高速缓存
├─────────────────────────────────────┤
│         主存储器 (DRAM)              │ ← 内存
├─────────────────────────────────────┤
│       辅助存储器 (磁盘/SSD)          │ ← 外存
└─────────────────────────────────────┘
     速度递减 ←——————→ 容量递增
```

### 4.2 主存储器（内存）

**DRAM（动态RAM）：**
- 需周期性刷新
- 成本低，容量大
- 速度较慢
- 用于主存

**SRAM（静态RAM）：**
- 无需刷新
- 成本高，容量小
- 速度快
- 用于Cache

### 4.3 辅助存储器

| 类型 | 特点 |
|------|------|
| 磁盘（HDD） | 容量大，成本低，速度慢 |
| 固态硬盘（SSD） | 容量大，速度快，成本高 |
| 磁带 | 顺序访问，用于备份 |

### 4.4 存储器的性能指标

- **存储容量**：存储二进制信息的总量
- **存取时间**：从启动访问到完成所需时间
- **存取周期**：连续两次独立访问的最小时间间隔
- **带宽**：单位时间内传输的数据量

**考试重点：** 存取周期 > 存取时间

## 5. 流水线技术

### 5.1 流水线的基本概念

流水线是将一个任务分解为多个子任务，每个子任务由专门的处理单元执行。

**流水线周期：** 各子任务中最长执行时间

### 5.2 流水线的性能指标

**1. 吞吐率（TP）**

$$TP = \frac{n}{T_k} = \frac{n}{(k+n-1)\Delta t}$$

其中：
- n：任务数
- k：流水级数
- Δt：时钟周期

**最大吞吐率：** $TP_{max} = \frac{1}{\Delta t}$

**2. 加速比（S）**

$$S = \frac{T_0}{T_k} = \frac{kn}{k+n-1}$$

**3. 效率（E）**

$$E = \frac{n \cdot k \cdot \Delta t}{(k+n-1)\Delta t \cdot k} = \frac{n}{k+n-1}$$

**考试重点：** 当n>>k时，效率趋近于100%，但永远达不到。

### 5.3 流水线分类

| 分类标准 | 类型 |
|----------|------|
| 按功能 | 单功能流水线、多功能流水线 |
| 按连接方式 | 静态流水线、动态流水线 |
| 按处理对象 | 标量流水线、向量流水线 |
| 按完成顺序 | 顺序流水线、乱序流水线 |

### 5.4 流水线相关问题

| 问题 | 描述 | 解决方案 |
|------|------|----------|
| 结构冒险 | 硬件资源冲突 | 增加资源 |
| 数据冒险 | 数据依赖 | 前推、暂停 |
| 控制冒险 | 分支指令影响 | 分支预测 |

## 6. 总线系统

### 6.1 总线的分类

| 分类 | 说明 | 示例 |
|------|------|------|
| 内部总线 | CPU内部连接 | 寄存器间总线 |
| 系统总线 | 计算机各部件连接 | ISA、EISA、PCI |
| 扩展总线 | 与外围设备连接 | USB、SCSI |

### 6.2 系统总线组成

| 总线类型 | 功能 |
|----------|------|
| 数据总线 | 传输数据（双向） |
| 地址总线 | 传输地址（单向） |
| 控制总线 | 传输控制信号 |

**考试重点：**
- 地址总线宽度决定寻址空间：n位地址 → 2^n字节
- 数据总线宽度决定传输能力
- 控制总线传输控制信号和时钟信号

### 6.3 总线仲裁

| 方式 | 特点 |
|------|------|
| 集中式仲裁 | 中央仲裁器统一仲裁 |
| 分布式仲裁 | 各设备仲裁逻辑分布 |

**集中式仲裁包括：**
- 链式查询方式
- 计数器定时查询方式
- 独立请求方式

## 7. Cache技术

### 7.1 Cache的基本原理

Cache位于CPU和主存之间，用于解决CPU与主存速度不匹配问题。

**命中率公式：**

$$h = \frac{N_c}{N_c + N_m}$$

其中：
- Nc：Cache命中次数
- Nm：主存访问次数

**平均访问时间：**

$$t_a = h \cdot t_c + (1-h) \cdot t_m$$

其中：
- tc：Cache访问时间
- tm：主存访问时间

### 7.2 Cache的映射方式

| 方式 | 特点 | 优缺点 |
|------|------|--------|
| 直接映射 | 主存块固定对应Cache行 | 简单，冲突率高 |
| 全相联映射 | 主存块可放任意位置 | 灵活，访问速度慢 |
| 组相联映射 | 组内全相联，组间直接映射 | 平衡方案 |

**考试重点：** 组相联是直接映射和全相联的折中方案，现代Cache常用。

### 7.3 Cache的写策略

| 策略 | 特点 | 问题 |
|------|------|------|
| 写直达 | 同时写Cache和主存 | 速度慢 |
| 写回 | 只写Cache，主存延迟写 | 数据不一致 |

**写分配法：** 写操作缺失时，先调入Cache再更新
**非写分配法：** 写操作缺失时，直接写主存，不调入Cache

## 8. 可靠性与性能评估

### 8.1 可靠性指标

**1. 失效率（λ）**

单位时间内系统发生故障的概率。

**2. 平均无故障时间（MTBF）**

$$MTBF = \frac{1}{\lambda}$$

**3. 可用性（A）**

$$A = \frac{MTBF}{MTBF + MTTR}$$

其中MTTR为平均修复时间。

### 8.2 系统的可靠性计算

**1. 串联系统**

$$R = R_1 \times R_2 \times ... \times R_n$$

所有子系统可靠度相乘。

**2. 并联系统**

$$R = 1 - (1-R_1)(1-R_2)...(1-R_n)$$

至少有一个子系统工作即可。

**考试重点：** 串联系统可靠性最低，并联系统可靠性最高。

### 8.3 阿姆达尔定律（重要考点）

$$S = \frac{1}{(1-f) + \frac{f}{k}}$$

其中：
- S：系统加速比
- f：可改进部分占系统总时间的比例
- k：改进部分的加速比

**考试重点：** 系统整体加速比取决于可改进比例和改进加速比的乘积。

### 8.4 性能评估方法

| 方法 | 特点 |
|------|------|
| 时钟频率法 | 简单，但不全面 |
| 指令执行速度法 | MIPs、MFLOPS |
| 等效指令速度法 | 考虑指令混合比例 |
| 综合评估法 | Amdahl定律 |

## 核心考点速记

1. **Flynn分类**：SISD/SIMD/MISD/MIMD
2. **CISC vs RISC**：RISC指令少、定长、多寄存器
3. **寻址速度**：立即 > 寄存器 > 直接 > 寄存器间接 > 间接
4. **流水线**：吞吐率取决于最长阶段
5. **Cache命中率**：h越大，平均访问时间越小
6. **可靠性**：串联最低，并联最高
7. **Amdahl定律**：S = 1/((1-f)+f/k)
