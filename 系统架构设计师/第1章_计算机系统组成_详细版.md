```

│                        用户应用层                            │
│              （办公软件、浏览器、数据库应用等）                  │
├─────────────────────────────────────────────────────────────┤
│                       应用软件层                              │
│               （开发工具、数据库管理系统等）                    │
├─────────────────────────────────────────────────────────────┤
│                       系统软件层                              │
│           （操作系统、编译程序、网络协议栈等）                   │
├─────────────────────────────────────────────────────────────┤
│                       支撑软件层                              │
│               （汇编器、链接器、加载器等）                      │
├─────────────────────────────────────────────────────────────┤
│                     硬件抽象层（HAL）                          │
│                   （设备驱动程序接口）                         │
├─────────────────────────────────────────────────────────────┤
│                       硬件层                                  │
│     （CPU、内存、I/O控制器、存储设备、外设等物理组件）            │
└─────────────────────────────────────────────────────────────┘
```

### 1.1.2 软硬件的等价性原理

**核心原理：** 硬件和软件在逻辑功能上是等价的。
- 某些功能既可以用硬件实现，也可以用软件实现
- 硬件实现：速度快，但缺乏灵活性
- 软件实现：灵活、可修改，但速度较慢
- **考试重点**：理解软硬件等价性对于理解系统架构设计至关重要

### 1.1.3 计算机语言层次

| 层次 | 语言类型 | 说明 |
|------|----------|------|
| 第1层 | 机器语言 | CPU能直接执行的二进制指令 |
| 第2层 | 汇编语言 | 用助记符表示的机器指令 |
| 第3层 | 高级语言 | C、Java、Python等 |
| 第4层 | 自然语言 | 人类日常用语（未来发展目标） |

**翻译程序分类：**
- **编译程序**：将高级语言全部翻译成目标代码（代表性语言：C/C++、Pascal）
- **解释程序**：逐句翻译执行，不产生目标代码（代表性语言：Python、JavaScript）
- **汇编程序**：将汇编语言翻译成机器语言

## 1.2 计算机硬件组成（五大部件）

### 1.2.1 冯·诺依曼体系结构

**核心特点：**
1. 计算机由五大部件组成
2. 采用二进制表示数据和指令
3. **存储程序原理**：程序和数据预先存放在存储器中
4. 指令顺序执行，由程序计数器控制

```
                    ┌─────────┐
                    │ 控制器  │
                    └────┬────┘
                         │
              ┌──────────┼──────────┐
              │          │          │
         ┌────▼────┐ ┌────▼────┐ ┌────▼────┐
         │ 运算器  │ │ 存储器  │ │ 输入设备│
         └────┬────┘ └────┬────┘ └─────────┘
              │          │
         ┌────▼──────────▼────┐
         │     输出设备        │
         └────────────────────┘
```

**考试重点：** 冯·诺依曼体系是现代计算机的基础，目前绝大多数计算机都是冯·诺依曼机。

### 1.2.2 五大部件详解

#### （1）运算器

**功能：** 执行所有的算术运算和逻辑运算。

| 组件 | 全称 | 作用 |
|------|------|------|
| ALU | Arithmetic Logic Unit（算术逻辑单元） | 核心执行部件，进行加减乘除、逻辑运算 |
| AC | Accumulator（累加器） | 存放运算结果或操作数 |
| DR | Data Register（数据缓冲寄存器） | 暂时存放从主存读/写的数据 |
| PSW | Program Status Word（程序状态字） | 记录运算结果的状态标志（零标志、进位标志、溢出标志等） |

**ALU的工作原理：**
```
输入1 ──┐
        ├──→ [ALU] ──→ 输出结果
输入2 ──┘     │
              ↓
         [PSW] ──→ 状态标志位
```

**状态标志位说明：**
- **零标志（Z）**：运算结果为0时置1
- **进位标志（C）**：无符号数运算产生进位/借位时置1
- **溢出标志（V）**：有符号数运算产生溢出时置1
- **符号标志（N）**：运算结果为负时置1

#### （2）控制器

**功能：** 协调控制计算机各部件正确执行指令。

| 组件 | 全称 | 作用 |
|------|------|------|
| PC | Program Counter（程序计数器） | 存放下一条指令的地址，指令执行后自动加1 |
| IR | Instruction Register（指令寄存器） | 存放当前正在执行的指令 |
| CU | Control Unit（控制单元） | 解释指令码，产生控制信号 |
| ID | Instruction Decoder（指令译码器） | 译码操作码，确定指令类型 |
| AR | Address Register（地址寄存器） | 暂时存放访问内存的地址 |

**指令执行过程（考试重点）：**

```
取指周期：
┌─────────────────────────────────────────────────────────┐
│  1. PC → AR：将从PC取出的指令地址送到地址寄存器          │
│  2. 存储器读取：将指令从内存读到数据寄存器                │
│  3. DR → IR：将指令送到指令寄存器                        │
│  4. PC+1：PC自动加1，指向下一条指令                       │
│  5. ID译码：指令译码器分析指令类型                       │
└─────────────────────────────────────────────────────────┘

执行周期（根据指令类型不同而不同）：
┌─────────────────────────────────────────────────────────┐
│  例：ADD R1, R2  （加法指令）                             │
│  1. 从寄存器取操作数                                     │
│  2. 送ALU进行运算                                        │
│  3. 结果送目标寄存器                                     │
│  4. 更新PSW状态标志                                      │
└─────────────────────────────────────────────────────────┘
```

**考试重点：**
- **PC自动加1**：每执行完一条指令，PC自动加1（对于多字节指令，加相应字节数）
- **PC的初始值**：程序开始执行前，PC指向程序入口地址
- **分支/跳转指令**：会修改PC的值，使其指向目标地址

#### （3）存储器

**功能：** 存储程序和数据。

| 分类 | 位置 | 特点 | 典型代表 |
|------|------|------|----------|
| 寄存器 | CPU内部 | 速度最快，容量最小 | 通用寄存器、专用寄存器 |
| Cache | CPU与主存之间 | 高速缓存 | L1/L2/L3 Cache |
| 主存 | 主板上 | 内存条 | DRAM |
| 辅存 | 机箱内/外 | 永久存储 | 硬盘、SSD |

**存储单元的组成：**
- 每个存储单元存放一个字节（8bit）或一个字
- 每个存储单元有唯一的地址
- 地址从0开始编号

**重要概念：**
- **字（Word）**：计算机一次能处理的二进制位数，常见有32位（4字节）、64位（8字节）
- **字节（Byte）**：基本存储单位，1字节=8位
- **位（bit）**：最小存储单位，只能存放0或1

**例题：** 一台计算机按字节寻址，其内存地址空间为4GB，问地址总线需要多少位？

**解析：** 4GB = 2^32字节，所以需要32位地址总线。

#### （4）输入设备

**功能：** 将外部信息输入计算机。

| 设备 | 输入形式 | 特点 |
|------|----------|------|
| 键盘 | 字符/数字 | 最基本的输入设备 |
| 鼠标 | 位置/点击 | 图形界面的主要输入 |
| 扫描仪 | 图像/文字 | 纸质文档数字化 |
| 触摸屏 | 触摸位置 | 移动设备主流输入 |
| 条码阅读器 | 条码数据 | 商业系统应用 |

#### （5）输出设备

**功能：** 将计算机处理结果输出。

| 设备 | 输出形式 | 特点 |
|------|----------|------|
| 显示器 | 图形/文字/视频 | 最重要的输出设备 |
| 打印机 | 纸质文档 | 票据、报表输出 |
| 音响 | 声音 | 多媒体输出 |
| 投影仪 | 图像 | 大屏展示 |

### 1.3.2 CPU的工作周期

**时钟周期（Clock Cycle）：**
- CPU最基本的的时间单位
- 由时钟脉冲信号控制
- 时钟频率 = 1/时钟周期（单位：Hz）

**机器周期（CPU Cycle）：**
- 完成一个基本操作的时间
- 一个机器周期包含若干个时钟周期
- 访问内存一次需要1个机器周期

**指令周期（Instruction Cycle）：**
- 执行一条指令所需的时间
- 一个指令周期包含若干个机器周期
- 不同指令的指令周期可能不同

**关系图：**
```
┌─────────────────────────────────────────────────────────────┐
│                     指令周期                                 │
│  ┌─────────────────┐      ┌─────────────────┐             │
│  │    取指周期      │  →   │    执行周期       │             │
│  │   (Fetch)       │      │   (Execute)     │             │
│  └────────┬────────┘      └────────┬────────┘             │
│           │                        │                       │
│           ▼                        ▼                       │
│  ┌─────────────────────────────────────────────────────┐  │
│  │                    机器周期                           │  │
│  │  ┌────┐ ┌────┐ ┌────┐ ┌────┐ ┌────┐ ┌────┐           │  │
│  │  │ T1 │ │ T2 │ │ T3 │ │ T4 │ │ T5 │ │ T6 │ ...       │  │
│  │  └────┘ └────┘ └────┘ └────┘ └────┘ └────┘           │  │
│  │                    时钟周期                           │  │
│  └─────────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────────┘
```

### 1.3.3 指令的执行流

**程序运行的本质：**
```
主存中的程序 → 取指 → 译码 → 执行 → 结果写回 → 下一条指令...
```

**考试重点：** CPU的工作就是不断地取指令、执行指令，周而复始，直到程序结束。

### 1.5.2 原码、反码、补码（常考）

| 码制 | 正数 | 负数 | 范围 | 用途 |
|------|------|------|------|------|
| 原码 | 符号位0，数值位不变 | 符号位1，数值位不变 | -127~+127 | 直观 |
| 反码 | 与原码相同 | 符号位1，数值位各位取反 | -127~+127 | 过渡 |
| 补码 | 与原码相同 | 符号位1，数值位取反加1 | -128~+127 | 主流 |

**考试重点：**
- 计算机中**有符号数用补码表示**
- 补码的特点：加法和减法可以统一为加法运算
- 8位补码表示范围：-128 ~ +127（注意：-128没有对应的原码和反码）
- 特殊数0的补码表示是唯一的：00000000

**快速转换方法：**
```
十进制 → 补码：
1. 正数：直接转二进制
2. 负数：先求绝对值的二进制，然后取反加1

补码 → 十进制：
1. 看符号位（最高位）
2. 若为0，直接转十进制
3. 若为1：取反加1，然后转十进制，再加负号
```

### 1.5.3 定点数与浮点数

**定点数：** 小数点位置固定
- 定点整数：小数点在最低位之后
- 定点小数：小数点在符号位之后

**浮点数：** 由阶码和尾码组成
```
浮点数 = 尾数 × 基数^阶码
       = M × R^E

其中：
- M：尾数（可正可负）
- R：基数（通常为2）
- E：阶码（可正可负）
```

**IEEE 754浮点数标准（考试重点）：**

| 格式 | 总位数 | 符号位 | 阶码位 | 尾数位 |
|------|--------|--------|--------|--------|
| 单精度 | 32位 | 1位 | 8位 | 23位 |
| 双精度 | 64位 | 1位 | 11位 | 52位 |

**例题：** IEEE 754单精度浮点数 0 10000001 10000000000000000000000 表示的十进制数是多少？

**解析：**
1. 符号位=0，正数
2. 阶码=10000001=129，偏移量=127，实际阶码=129-127=2
3. 尾数=1.1（隐含最高位1）=1.5
4. 结果=1.5 × 2² = 6.0

### 1.6.2 典型例题解析

**【例题1】** 冯·诺依曼体系结构的核心思想是？

A. 采用二进制表示数据
B. 存储程序和程序控制
C. 计算机由五大部件组成
D. 运算器和控制器分离

**答案：B**

**解析：** 冯·诺依曼体系的核心是"存储程序和程序控制"，即程序预先存放在存储器中，CPU按顺序执行。A、C、D也是其特点，但不是核心思想。

---

**【例题2】** 在CPU中，程序计数器（PC）的作用是？

A. 存放当前正在执行的指令
B. 存放指令的操作码
C. 存放下一条要执行的指令地址
D. 存放运算结果

**答案：C**

**解析：** PC（Program Counter）存放下一条要执行的指令地址。IR存放当前指令，AC存放运算结果。

**【例题3】** 8位补码能表示的整数范围是？

A. -127 ~ +127
B. -128 ~ +127
C. -255 ~ +255
D. -128 ~ +128

**答案：B**

**解析：** 8位补码表示范围是-128 ~ +127。注意-128的补码是10000000，没有对应的原码和反码。

**【例题4】** 计算机系统的软硬件等价性原理是指？

A. 硬件和软件功能等价
B. 硬件和软件价格等价
C. 硬件和软件速度等价
D. 硬件和软件可靠性等价

**答案：A**

**解析：** 软硬件等价性原理指同样的功能既可以用硬件实现，也可以用软件实现，在逻辑功能上是等价的。
