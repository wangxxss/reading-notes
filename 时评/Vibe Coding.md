
2025年2月， Andrej Karpathy（OpenAI 联合创始人、前特斯拉 AI 总监）发了一条推特：

> "我最近写了好多代码，但基本不自己敲。主要就是坐在那里盯着屏幕，然后对 AI 说'帮我改一下这个'、'啊不对，应该是这样'。"

他给这种新的编程方式起了一个名字——**Vibe Coding**。

这条推文瞬间刷屏。因为每个正在使用 Cursor、Copilot、Claude Code 的开发者，都从中看到了自己。

------

## 一、什么是 Vibe Coding？

### 1.1 一句话定义

**Vibe Coding**，就是用"感觉"来编程。

不再是逐行敲代码、编译、调试、再修改的传统流程，而是：

- **用自然语言描述需求**："我要一个用户登录页面，支持手机号和微信登录"
- **AI 生成代码**：模型返回完整的实现
- **你审阅、调整、再描述**："密码输入框加个显示/隐藏功能"、"登录按钮颜色换一下"
- **循环迭代**：直到"感觉对了"为止

**整个过程，你写的不是代码，是意图。**

### 1.2 从传统编程到 Vibe Coding

|     维度     |       传统编程       |       Vibe Coding        |
| :----------: | :------------------: | :----------------------: |
| **工作方式** |    手写代码为主      | 自然语言描述 + AI 生成   |
| **核心技能** | 语法、API、调试      | 需求描述、架构设计、代码审查 |
| **编程语言** | 必须精通             | 仍需掌握，但书写量大幅减少 |
| **产出方式** |     全部手写         | AI 生成初稿 + 人工审查修改 |
| **效率变化** |     线性增长         | 在重复性工作上有显著提升   |

### 1.3 Vibe Coding 不是"不写代码"

一个常见的误解是：Vibe Coding 就是让 AI 写所有代码，程序员失业了。

**不是的。**

Vibe Coding 的本质是**编程焦点的转移**——从"怎么写"转移到"写什么"和"为什么写"。

你仍然需要：

- **理解系统的整体架构**——AI 无法替你做架构决策
- **审查 AI 生成的代码质量**——这要求你对编程语言有扎实的理解
- **处理复杂的边缘情况**——AI 倾向于生成"happy path"代码
- **调试你没亲手写的代码**——这比调试自己写的代码更难，不是更容易
- **把多个 AI 生成的模块拼成完整系统**——集成永远是最难的部分

**Vibe Coding 不是降低了对工程师的要求，而是改变了要求的重心。**

## 二、为什么 Vibe Coding 值得关注？

### 2.1 AI 编程能力已达拐点

2025-2026年，AI 在代码生成领域发生了质变：

- **代码补全** → **代码生成** → **自主编程（Agent）**
- Claude Code、Cursor、Windsurf 等工具已经可以独立完成中等复杂度的开发任务
- 国产模型（GLM-5、Qwen-3.5、Kimi-K2.5）在中文代码生成上表现优秀

**用 AI 写代码，已经不是"能不能"的问题，而是"怎么用得更好"的问题。**

### 2.2 真正的价值：释放工程师的高阶能力

一个典型开发任务的时间分布大致如下：

|   环节   | 大致占比 |         说明         |
| :------: | :------: | :------------------: |
| 需求理解 |  ~30%   | 理解用户真正想要什么 |
| 架构设计 |  ~20%   |   如何组织系统结构   |
| 编码实现 |  ~25%   |      实际写代码      |
| 测试调试 |  ~15%   |    发现和修复问题    |
| 部署运维 |  ~10%   |   上线、监控、维护   |

> 注：这个比例因项目类型、团队规模和开发阶段不同会有较大差异，这里取的是一个粗略的中值。

Vibe Coding 主要加速的是"编码实现"这 25%。即使提升 3-5 倍，**对整体效率的提升大约在 20%-40% 左右**——显著，但不是魔法。

真正的价值不在于"快了多少"，而在于：**省下来的时间，可以投入到需求理解、架构设计这些更高价值的环节中。**

### 2.3 行业已经在行动

全球顶级科技公司已经全面拥抱 AI 辅助编程：

- **Google**：超过 25% 的新代码由 AI 生成
- **GitHub**：Copilot 用户编码效率平均提升 55%
- **国内大厂**：阿里、字节、腾讯等已推出各自的 AI 编程方案

这些数据说明 AI 辅助编程已经是行业共识，而不是少数人的实验。

## 三、如何开始 Vibe Coding？

### 3.1 第一步：选择你的 AI 编程工具

**入门推荐（免费/低价）：**

- **Cursor**（免费版）：目前体验最好的 AI IDE
- **VS Code + Continue/Cline**：完全免费，插件生态丰富
- **Kimi Code**：国产方案，中文体验好

**进阶推荐（付费）：**

- **Cursor Pro**（$20/月）：智能程度更高
- **Claude Code**：适合复杂项目，终端交互模式
- **各平台 AI 编程方案**：详见本号往期文章

### 3.2 第二步：学会"说人话"

Vibe Coding 的核心技能不是编程，而是**精确描述需求**。

**差的提示词：**

> "帮我写一个登录页面"

**好的提示词：**

> "创建一个用户登录页面，使用 React + Tailwind CSS。需要支持两种登录方式：手机号+验证码、微信扫码。页面布局要求：左侧品牌展示区（深色背景、Logo、Slogan），右侧登录表单区（白色背景、表单居中）。手机号输入框需要格式校验（11位数字），验证码按钮需要60秒倒计时。整体风格简洁现代，移动端适配。"

**好的提示词 = 具体的需求 + 明确的约束 + 完整的上下文**

补充一个实战技巧：**把项目的技术栈、代码规范、目录结构等信息整理成文档（如 CLAUDE.md 或 .cursorrules），让 AI 自动读取，比每次手写上下文高效得多。**

### 3.3 第三步：建立 AI 代码审查习惯

AI 生成的代码不等于正确的代码。你需要重点关注：

1. **功能正确性**：AI 是否理解了你的需求？逻辑是否正确？
2. **安全审查**：有没有 SQL 注入、XSS、硬编码密钥等安全漏洞？
3. **性能审查**：有没有 N+1 查询、不必要的重渲染、内存泄漏等问题？
4. **边界检查**：空值、异常输入、并发场景是否处理了？
5. **可维护性**：代码是否清晰可读？三个月后你（或你的同事）能看懂吗？
6. **架构一致性**：代码风格和项目结构是否统一？有没有引入不必要的依赖？

**AI 是你的超级实习生——干活很快，但需要你把关。而把关的前提是：你自己得有足够的技术判断力。**

### 3.4 第四步：循序渐进

不需要一步到位，建议按以下节奏逐步深入：

- **第 1-2 周**：用 AI 写重复性代码（CRUD、表单、配置文件、单元测试）
- **第 3-4 周**：用 AI 生成完整功能模块，自己审查和集成
- **第 5-8 周**：用 AI 搭建项目骨架，自己把控核心业务逻辑和架构
- **第 8 周以后**：尝试用 AI Agent 处理多步骤开发任务

> 提示：每个阶段都要有意识地总结——哪些场景 AI 做得好，哪些场景你不得不自己上。这个经验比工具本身更有价值。

## 四、AI 编程时代，工程师如何升级技能？

### 4.1 技能重心的转移

|    能力维度    | 传统时代 | AI 时代 |         变化趋势         |
| :------------: | :------: | :-----: | :----------------------: |
|    编码速度    |   核心   |  重要   | ↓ 不再是核心竞争力，但仍需具备 |
|    算法能力    |   重要   |  重要   | → 基本持平               |
|    系统设计    |   重要   |  核心   | ↑ 重要性上升             |
|    产品思维    |   加分   |  核心   | ↑↑ 大幅上升             |
|   AI 协作力   |   无     |  核心   | ↑↑↑ 全新的核心技能       |
| 代码审查能力   |   重要   |  核心   | ↑ 从审查人写的代码，变成审查 AI 写的代码 |
|    架构思维    |   重要   |  核心   | ↑↑ 大幅上升             |

### 4.2 五个核心升级方向

**1. 从"实现者"升级为"设计者"**

AI 能帮你写代码，但无法替你做架构决策。

你需要强化：

- 系统架构设计（微服务、事件驱动、CQRS...）
- 数据建模（如何设计数据结构来支撑业务）
- 技术选型（什么场景用什么方案最优）

**2. 培养"AI 协作力"**

这是 AI 时代最重要的新技能：

- **Prompt Engineering**：学会高效地描述需求
- **代码审查能力**：快速识别 AI 代码的问题——AI 生成的代码往往"看起来对"但逻辑上有微妙错误
- **上下文管理**：知道该给 AI 提供多少信息，太少则生成质量差，太多则超出窗口限制
- **迭代思维**：通过多轮对话逐步逼近理想结果，而不是期望一次搞定

**3. 强化产品思维**

当编码不再是瓶颈，你与用户之间的距离就缩短了。

- 理解用户痛点，而不是只看需求文档
- 思考"为什么要做这个功能"，而不只是"怎么做"
- 用最小成本验证想法，快速迭代

**4. 拥抱全栈能力**

AI 让跨领域开发的门槛降低了：

- 前端工程师可以更容易地写后端 API
- 后端工程师可以更快地搭建前端页面
- 所有人都可以更轻松地处理 DevOps 配置

但要注意：**门槛降低不等于可以不懂。** AI 帮你生成了 Kubernetes 配置，你至少要能看懂它在做什么，否则出了问题你束手无策。

**5. 持续学习，保持手感**

AI 技术迭代极快，今天的最佳实践可能三个月后就过时了：

- 关注 AI 编程工具的更新动态
- **定期手写代码**——不要完全依赖 AI，保持基本功
- 建立个人的提示词库和最佳实践文档
- 与同行交流，分享经验

### 4.3 能力模型

未来工程师的核心竞争力，从下往上依次递增：

```
 ┌───────────────────────────────────────┐
 │  战略层 │ 产品思维、商业理解、用户洞察  │  ← 最难被替代
 ├───────────────────────────────────────┤
 │  设计层 │ 架构设计、技术选型、数据建模  │  ← 核心竞争力
 ├───────────────────────────────────────┤
 │  协作层 │ AI 交互、代码审查、上下文管理 │  ← 新兴关键技能
 ├───────────────────────────────────────┤
 │  执行层 │ 编码实现、调试（AI 大幅加速） │  ← 仍需具备，但不再是重心
 └───────────────────────────────────────┘
```

**每一层都不可或缺，但越往上，越难被 AI 替代，也越值得你投入时间。**

## 五、Vibe Coding 的局限与现实挑战

作为工程师，我们需要诚实地面对 Vibe Coding 目前的局限。**只有清楚边界在哪，才能用好它。**

### 5.1 不适合 Vibe Coding 的场景

- **高性能计算**：算法优化需要精确到指令级的调优，AI 生成的代码通常达不到这个精度
- **安全敏感领域**：金融、医疗等需要严格代码审计的场景，AI 生成的代码必须经过额外的安全审查流程
- **创新性算法**：AI 擅长组合已有模式，不擅长发明全新算法
- **复杂系统调试**：分布式系统的疑难杂症，往往需要对系统的深度理解，而不仅仅是代码层面的修改

### 5.2 工程实践中的真实挑战

- **技术债务**：AI 倾向于生成"能跑"的代码，而非"好维护"的代码。如果不加审查，技术债务会快速累积
- **代码一致性**：AI 每次生成的代码风格可能不同，在团队协作中容易造成风格混乱
- **调试困难**：当 AI 生成的代码出 bug 时，你需要理解一段"别人写的代码"——而这个"别人"没法回答你的问题
- **上下文限制**：AI 模型有上下文窗口限制，对大型项目无法一次性理解全貌
- **成本考量**：高质量 AI 编程工具（Cursor Pro、Claude Code）有持续的订阅成本，团队规模化使用需要考虑预算

### 5.3 警惕"AI 依赖症"

- **不要丧失基础编程能力**——你需要能看懂、修改、甚至重写 AI 的代码
- **不要丧失独立思考**——AI 会给你一个"看起来合理"的答案，但判断对错的是你
- **不要丧失调试能力**——AI 解决不了的问题，你得能接手
- **不要丧失学习动力**——"反正 AI 会写"是一个危险的想法

## 写在最后

Vibe Coding 不是一种编程技术，而是一种**工作方式的演进**。

它意味着：

- 从"我写代码"到"我指导 AI 写代码，然后审查和集成"
- 从"关注实现细节"到"更多精力投入架构和业务"
- 从"单一技能深耕"到"跨领域能力拓展"
- 从"独立工作"到"人机协作"

但请记住：**工具在变，工程师的核心价值没变——理解问题、设计方案、交付可靠的软件。** Vibe Coding 只是让你在这条路上跑得更快，而不是换了一条路。

现在就开始尝试吧。从一个小功能开始，用 AI 生成，自己审查，逐步建立你的 Vibe Coding 工作流。
