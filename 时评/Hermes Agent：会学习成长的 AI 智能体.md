# Hermes Agent：会学习的 AI 助手，这次是真的记得你

---

你遇到过这种情况吗？

让 AI 帮你处理个项目，头一天还挺聪明，问啥都能答上来。第二天打开对话框，它一脸茫然地看着你："抱歉，我不太清楚您之前说的项目呢。"

第三次、第十次，依然如故。每次都像在跟一个Goldfish（金鱼）对话——七秒记忆，重来一遍。

大多数 AI 助手都这样。不是它们不想记住你，是它们的"记忆"本质上是把对话塞进提示词，下一次会话一开始，那段文本就没了。你以为你在跟同一个 AI 对话，其实每次都是个全新的陌生人。

Hermes Agent 想解决这个问题。

---

## 这个 Agent 有点不一样

Hermes Agent 来自 Nous Research。

如果你没听过这个名字，那你应该听说过他们的 Hermes 系列模型——开源社区里工具调用能力最强的模型之一。2024 年他们发布 Hermes 3，2025 年（预计）发布 Hermes 4，支持混合推理，还搞了个自动数据生成系统，挺能折腾的。

Hermes Agent 是他们把模型能力和基础设施捏在一起的产品。官方 slogan 是"随你成长的 Agent"——听起来像营销话术，但读进去会发现，这其实是它真正的架构逻辑。

---

## 它怎么记住你？

大多数 AI 工具是无状态的。Hermes Agent 不一样，它有自己的一套记忆体系，分三层。

**第一层：会话记忆**

每次对话结束，内容会被存进 SQLite 数据库，用 FTS5 做全文索引。下次再开对话，它可以主动搜索历史，把相关内容捞出来。

不是按时间线硬翻，而是找相关性。你问"上次那个 Bug 后来怎么修的"，它会搜，会总结，会把结论塞进上下文。

**第二层：知识记忆**

这一层更有意思。Agent 在对话过程中会自己判断：哪些东西值得长期保存？

你的偏好、项目的技术栈、你反复提到的上下文——它会提炼成结构化的记忆条目，持久存储。

它还有个叫 Honcho 的组件，做用户建模。不是简单存你说了什么，而是尝试理解你这个人。你是喜欢简洁回复还是详细解释？你是夜猫子还是早起党？这些会在记忆里慢慢沉淀。

**第三层：技能记忆**

这是 Hermes 最特别的设计。

你让它帮你完成一个复杂任务，比如"配个 Docker 环境部署 Next.js 项目"。它搞定之后，会自动把整个流程写成一个 Skill 文件——什么步骤、什么命令、什么坑，下次遇到类似需求直接调用。

更离谱的是，这个 Skill 在使用过程中还会自己优化。

举个例子：你第一次让它配 Docker 环境，它写了个 10 步的流程。过了两周让它再配一次，它自己发现"Ubuntu 22.04 下步骤 5 到 7 可以合并"，直接精简成 8 步，还自动标注了"某系统下无需执行步骤 5"。这就是它的自我优化——不是人告诉它改的，是它自己用出来的经验。

Skills 兼容 agentskills.io 社区标准，你可以分享自己写的技能，也可以用别人贡献的。

三层加起来，形成一个闭环。用得越多，它对你的了解越深，能帮你搞定的事情越多。

## 技术上的几个亮点

**模型配置很灵活**

支持 Nous Portal、OpenRouter（200+ 模型随便切）、OpenAI API，还有本地 vLLM。想换模型？一行命令的事：

```bash
hermes model
```

你可以让写代码用一个模型，画图用另一个，不强制绑定。

**多平台接入**

Telegram、Discord、Slack、WhatsApp、Signal、CLI，都在一个网关进程里管理。你在 Telegram 开始聊，换到 Discord 可以接着聊。

**六种运行后端**

本地、Docker、SSH、Daytona、Singularity、Modal。5 美元的 VPS 能跑，GPU 集群也能跑，闲置时几乎不花钱。

**40+ 内置工具**

网页搜索、终端命令、文件操作、图像生成、TTS、视觉——都是高频工具。比如让它帮你搜某个技术方案的对比，它直接调网页搜索；让它帮你批量重命名文件，它调文件操作；让它跑个部署脚本，它调终端命令。不需要自己写插件，常见需求它自己就能搞定。

## Hermes 和 OpenClaw，什么关系？

这是很多人关心的问题。

先说 OpenClaw。它是 PSPDFKit 创始人 Peter Steinberger 发起的项目，预计 2025 年达到爆发式增长（有数据显示月活开发者从 2024 年底的 3 万增长到 2025 年的 45 万）。GitHub 16 万+ stars，50+ 消息渠道支持，社区庞大，插件生态丰富。

Hermes Agent 和 OpenClaw 架构上很像——都是网关式设计，中央进程管消息渠道，路由到 AI 模型。定位也相似：让 AI 脱离聊天窗口，真正帮你干活。

但关键区别在这里：

OpenClaw 为团队复杂多 Agent 操作设计，社区大，插件多。Hermes Agent 为个人开发者设计，强调"越用越懂你"的学习闭环。

更有意思的是迁移支持。Hermes Agent 直接内置了从 OpenClaw 迁移的工具：

```bash
hermes claw migrate
```

你 OpenClaw 上的记忆、配置、对话历史，可以一键搬过来。Hermes 团队甚至写了份很长的迁移文档，生怕你搬过程中丢东西。

这意味着什么？Hermes 团队花了很大力气做从 OpenClaw 的完整迁移——内置迁移工具、写详细迁移文档、支持记忆和配置的一键搬迁。这种设计倾向，说明他们瞄准的是从 OpenClaw 迁移过来的个人开发者场景，而非团队协作场景。

当然，两者也可以一起跑。通过 ACP（Agent Communication Protocol）协议，可以让 Hermes 负责规划工作流，OpenClaw 负责执行，各取所长。

---

## 你该选哪个？

选 Hermes Agent 如果你：
- 想要一个"越用越懂你"的助手
- 对强化学习训练轨迹生成有需求
- 喜欢 Python 生态

选 OpenClaw 如果你：
- 需要 50+ 消息渠道支持
- 看重社区规模和插件生态
- 需要 Signal、Google Chat 等 Hermes 不支持的渠道
- 喜欢 TypeScript/Node.js 生态

Hermes 官方文档里有句话挺实在的："Hermes 不是要在 OpenClaw 的赛道上打败它。两者服务的是不同用户群。"

## 值得关注的理由

Hermes Agent 不是最成熟的 AI Agent 项目，但它在尝试解决一个真正重要的问题：怎么让 AI 记住你，并且越用越好。

不是靠更大的上下文窗口，是靠一套三层记忆架构和闭环学习机制。逻辑上跟大多数 Agent 产品拉开了差距。

它和 OpenClaw 是竞争也是互补。两者都在推动 AI 从"聊天机器人"变成"真正能帮你干活的智能体"，只是走了不同的技术路线。

如果你对 AI Agent 的进化方向感兴趣，Hermes Agent 值得放一个在 VPS 上跑跑看——用得越久，它越懂你。

官网（hermesagent.agency）提供快速部署指南，一行命令装好：
```bash
curl -fsSL https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.sh | bash
```
装完跑 `hermes setup`，五分钟就能开始聊。
