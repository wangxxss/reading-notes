Trae CN 是字节跳动推出的 AI 编程工具，对中文开发者比较友好。它集成了豆包、DeepSeek-R1、DeepSeek-V3 等模型，可以用自然语言交互来生成代码、构建项目和调试程序。

一、安装与配置

1.1 下载安装

访问 Trae CN 官网（https://www.trae.com.cn/download），下载对应系统的版本：

  • Windows：下载 .exe 安装程序
  • Mac：下载 .dmg 文件
  • Linux：下载 .tar.gz 文件

Windows 系统安装比较简单：双击下载的 .exe 文件，一路点"下一步"就行。安装路径不建议选 C 盘，除非 C 盘空间很充裕。

1.2 首次启动设置

第一次打开 Trae CN 会进入初始配置流程。选个喜欢的主题（暗色、亮色、深蓝三选一），语言选简体中文，然后点"继续"。

之前用 VS Code 或 Cursor 的话，可以点"从 VS Code 导入"或"从 Cursor 导入"，插件、设置、快捷键都会自动同步过来。想全新开始的直接点"跳过"。

1.3 命令行工具

点击"安装 trae 命令"可以在终端里快速调出 Trae。装好之后：

  • trae 快速打开 Trae CN
  • trae my-react-app 在 Trae CN 里打开指定项目

1.4 登录

AI 功能需要登录才能用。国内版支持手机号或掘金账号登录。

二、核心工作模式

Trae CN 有两种主要工作模式：Chat 模式和 Builder 模式。

2.1 Chat 模式：代码问答与辅助

Chat 模式是默认模式，适合代码问答、查错、优化这些场景。按 Command/Ctrl + U 打开侧边对话框，点左上角的"Chat"进入。

主要功能：

代码解释：选中代码后右键选"Ask Trae"，AI 会解释代码逻辑。也可以在对话框输入"解释这段代码"或直接敲 /explain，或者点代码文件顶部的"解释代码"按钮。

错误修复：把终端报错信息拖到对话框，AI 会给出修复建议。处理依赖问题或语法错误比较方便。

代码优化：输入"优化这段 Python 代码的运行效率"这类需求，AI 会给出具体方案。

技术问答：直接问技术问题就行，比如"React 怎么用 Redux"。

单元测试：选中代码后在对话框输入"生成单元测试"或敲 /test，也可以右键菜单或点文件顶部的"生成单测"按钮。

代码注释：选中代码后输入"生成注释"或敲 /doc，同样支持右键菜单和顶部按钮操作。

2.2 Builder 模式：从零构建项目

Builder 模式是 Trae CN 区别于传统 IDE 的核心特色，用自然语言描述需求，AI 自动生成完整项目。按 Command/Ctrl + U 打开侧边对话框，点左上角的"Builder"，再点"启用 Builder"，选 DeepSeek-R1 或 DeepSeek-V3 模型。

主要能力：

自然语言生成项目：直接说需求就行。比如"用 Python 写个贪吃蛇，蛇初始 3 节，吃食物变长，食物随机刷新，碰边界或自身就结束，帧率 15，用 pygame"。AI 会自动创建项目文件夹。

自动装依赖：点编辑器下方的运行按钮，AI 会自动安装需要的依赖。

多文件协调：需要添加功能时，AI 会同时修改多个关联文件。

实时预览：网页项目可以一键预览效果，不用另外开浏览器。

2.3 SOLO 模式：AI 主导的自动化开发

SOLO 模式在 2025 年 11 月推出，代表了 AI 编程工具的一个进化方向——从辅助写代码变成 AI 自主完成开发任务。这个模式下，AI 可以理解目标、管理上下文、调度工具，独立推进各个开发阶段，从需求到部署全链路自动化。

2.3.1 和 IDE 模式的区别

维度 IDE 模式 SOLO 模式
定位 AI 辅助开发 AI 自主开发
界面 对话 + 代码编辑器联动 三栏式：左侧对话、中间文件/编辑器、右侧工具面板
能力 生成/修改代码 多任务并行、自动规划、上下文压缩、自主调试部署
智能体 单一 AI 双智能体协同（SOLO Coder + SOLO Builder）

2.3.2 两个核心智能体

SOLO Coder：处理复杂任务的迭代。适合现有代码库的优化、Bug 修复、重构、跨模块变更、接口改造等场景。

SOLO Builder：从零开始落地新项目。适合全新应用搭建、需求转技术方案、快速原型开发等，从需求解析到代码开发再到发布上线，全流程覆盖。

简单来说：0-1 项目搭建用 Builder；已有项目的迭代、重构、修 Bug 用 Coder。

2.3.3 进入与切换

点左上角的"模式切换"按钮，选 SOLO。首次进入要选智能体类型（SOLO Coder 或 SOLO Builder），之后可以在顶部工具栏切换。输入项目名称和开发语言，点击"启动 SOLO 模式"。

2.3.4 Plan 模式

处理复杂任务时，建议先开 Plan 模式。勾选输入框下方的"开启 Plan 模式"，AI 会先输出开发计划，包含任务拆解、技术选型、步骤安排、预期结果等内容。看完后点"按此计划执行"，或者提修改意见。确认后 AI 按步骤推进。快捷键是 Option + P。

2.3.5 实时跟随

打开工具面板左上角的"实时跟随"按钮，系统会根据 AI 当前的工作阶段自动切换工具。比如 AI 在写 PRD 时自动打开文档工具，在写代码时自动切换到编辑器。

AI 处理任务时工具是只读状态。如果想手动操作，双击或滚动内容会退出实时跟随模式。需要干预时先关掉这个功能。

2.3.6 多任务管理

SOLO 支持同时管理多个任务，不用等一个完成再开始下一个。

2.3.7 工具面板与集成服务

SOLO 提供编辑器、文档、浏览器、终端等工具。国际版还集成了 Supabase（数据库）、Vercel（部署）、AI 服务和 Stripe（支付）。

2.3.8 Figma 导入

可以把 Figma 设计文件直接转成代码。选择整个 Frame 或单个组件（按钮、表单、卡片），发给 AI 就能生成对应代码。

2.3.9 自定义智能体

可以创建自定义智能体给 SOLO Coder 调用。在创建入口勾选即可。SOLO Coder 自己是主智能体，创建的是子智能体，只能被 Coder 调用，子智能体之间不能互相调用。Coder 内置了一个 Search 智能体。

2.3.10 计费

版本 SOLO 模式 说明
中国版 免费 完全免费
国际版 需 Pro 用户 按 Max 模式 Token 计费

SOLO 模式强制开 Max 模式。使用自己的自定义模型不额外收费，只消耗自己的 API 配额。

四、代码补全功能

4.1 自动代码补全

敲回车换行时，AI 会阅读当前代码并自动补全后续代码。也可以先写注释再回车，AI 会根据注释生成对应代码。

补全建议出现后：点"接受"或按 Tab 采纳全部；点"接收 Word"或 Ctrl + 右箭头逐字添加；按 Esc 清除。

4.2 Tab-Cue 智能预测

AI 会根据上一次的编辑内容预测下一个改动点。按 Control + Shift + Enter（macOS 是 Control + Shift + Enter，Windows 相同）主动触发补全，按 Tab 采纳。采纳后会跳到下一个改动点，继续按 Tab 采纳。

代码补全 Pro 默认开启，可以在设置里关掉。

六、进阶技巧

6.1 精准提问

选中代码后右键"Ask Trae"，比在对话框里粘贴代码片段效果更好，AI 能更准确地理解上下文。

6.2 文件夹级上下文

把整个项目文件夹拖进对话框，说"添加登录功能"，AI 会分析项目结构，同时修改多个文件（路由、样式、API），不只是单一文件。

6.3 历史回溯

点对话记录的"初始版本"可以一键回退代码，支持 diff 对比。

6.4 快捷键

  • Command/Ctrl + U：打开侧边对话框
  • Tab：采纳 AI 推荐的代码
  • Esc：清除推荐的代码
  • Ctrl + 右箭头：逐字接受推荐
  • Control + Shift + Enter：主动触发 Tab-Cue 补全

七、常见问题

启动报权限不足：Windows 用管理员身份运行终端；Mac/Linux 前面加 sudo。

页面空白：关掉浏览器隐私模式，检查 8080 端口是否被占用。

模型响应慢：在设置里切换 DeepSeek-R1 模型，速度通常比豆包快。

依赖安装失败：换 npm 国内源：

npm config set registry https://registry.npmmirror.com

八、总结

Trae CN 的核心理念是"说人话，自动写"。优点：全中文界面，响应快；文件夹级上下文理解比单文件强；内置预览调试；Chat 和 Builder 模式互补。

建议每天花半小时练一个小案例，一周左右就能从零做出完整项目。

官方文档：https://www.trae.com.cn/docs
